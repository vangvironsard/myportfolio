តើអ្វីទៅជា WEB 3.0?

អត្ថបទដោយ៖ ISTAD



មួយរយ:ក្រោយនេះ WEB 3.0 ត្រូវបានគេស្គាល់ថាជា generation (1.0, 2.0, 3.0) ថ្មីនៃការប្រើប្រាស់ internet។


មុននឹងស្វែងយល់អំពី WEB 3.0 យើងនឹងធ្វើការនិយាយអំពី ប្រវត្តិ របស់ WEB និង​ ការវិវត្តន៍នៃ Internet ជាមុនសិន។ WEB បានកើត នៅប្រទេស Switzerland ក្នុងឆ្នាំ 1989។ នៅក្នុងខែមីនា ថ្ងៃទី​១២ ឆ្នាំ 1989, អ្នកវិទ្យាសាស្ត្រកុំព្យូទ័រ ឈ្មោះ Tim Berners-Lee​ អាយុ ៣៣ឆ្នាំម្នាក់ បានដាក់ Propopsal ទៅកាន់ European Organization for Nuclear Research (CERN)។ នោះគឺជា WEB 1.0 ដែលបម្រើដល់ការ រក្សានិងផ្ទុកទុកនូវព័ត៌មានផ្សេងៗនៅលើ Internet ជាមួយនឹង hypertext។ ក្នុងឆ្នាំ 1990 website ដំបូងគេបង្អស់នៅក្នុងពិភពលោក បានអភិវឌ្ឍនិងដាក់ឱ្យប្រើប្រាស់នៅក្នុងឆ្នាំ 1993 ហើយបន្ទាប់មកនិងមាន web browserឈ្មោះ MOSAIC បានអភិវឌ្ឍសម្រាប់ប្រើប្រាស់ផងដែរ (កើតមុន Internet Explorer)។






WEB 1.0 សម្តៅទៅលើការកើតដំបូងនៃ World Wide Web evolution​ ដែលអាចឱ្យយើងធ្វើការស្វែងរក ព័ត៌មានក្នុងទម្រង់ web។ វាមានលក្ខណ:​​ ដូចជា៖

  • Static pages
  • Content គឺទាញយកមកពី server’s file system
  • Pages បង្កើតឡើងដោយប្រើប្រាស់ Server Side ឬ Common Gateway Interface (CGI)។
  • ​​Frame និង Table ត្រូវបានប្រើប្រាស់សម្រាប់ដាក់ទីតាំង និង តម្រៀបធាតុនៅលើ page។






WEB 2.0 ត្រូវបានគេឱ្យឈ្មោះថា Participative និង Social Web ដោយសារតែជាទូទៅ website ទាំងនោះផ្តោតទៅលើ​ user-generated content, participatory culture, usability សម្រាប់អ្នកប្រើប្រាស់។ ពាក់ព័ន្ធជាមួយ បច្ចេកវិទ្យា ដែលប្រើនៅក្នុងការ develop web 2.0 មានរួមចំណែកពី AJAX និង JavaScript Framework។ វាមានលក្ខណ:​​ ដូចជា៖

  • Dynamic content ដែលទទួលបានពីការបញ្ចូលរបស់អ្នកប្រើប្រាស់
  • ​​​​​ច្រោះព័ត៌មានដោយសេរី និងអនុញ្ញាតអ្នកប្រើប្រាសឱ្យទទួលបានទិន្នន័យឬ ព័ត៌មាន បានយ៉ាងងាយស្រួល
  • User participation សម្តៅលើចរន្តនៃព័ត៌មាន ទទួលបានពីអ្នកប្រើប្រាស់ឬ ម្ចាស់ផ្ទាល់










WEB 3.0 បានក្លាយជា vision នៃ better internet។ ផ្នែកស្នូលរបស់វា ប្រើប្រាស់ blockchain, cryptocurrencies និង​ NFTs សម្រាប់ជាផលប្រយោជន៍ទៅកាន់អ្នកប្រើប្រាស់ក្នុងទម្រង់ជា​ ownership។ ជាមួយគ្នានេះដែរ Web3 មាន principles សំខាន់ៗដូចជា៖

  • Web3 គឺ Decentralized: ជំនួសនូវការប្រើប្រាស់ access internet ទៅកាន់ service ដែលគ្រប់គ្រងដោយក្រុមហ៊ុនណាមួយ (Google, Facebook, ...) decentralized ផ្តល់ ownership ទៅកាន់អ្នកបង្កើតនិងអ្នកប្រើប្រាស់
  • ​​​​​Web3 គឺ permissionless: អ្នកប្រើប្រាស់ទាំងអស់មានសិទ្ធិ access ស្មើៗគ្នា។
  • Web3 មាន native payments: វាប្រើប្រាស់ cryptocurrency សម្រាប់ការចាយវាយ បង់ថ្លៃ តាមអនឡាញ ជំនួសឱ្យការប្រើប្រាស់សេវាធនាគារឬសេវាផ្សេងៗ
  • Web3 គឺ​ Trustless: វាដំណើរការដោយប្រើប្រាស់ incentive និង economic mechanism ជំនួយឱ្យការប្រើប្រាស់ ឈ្មួញកណ្តាល។




ប្រសិនបើលក្ខណ:ពិសេសរបស់ Web3 នៅតែពិបាកក្នុងការយល់ យើងនឹងធ្វើការពន្យល់អោយមានភាពសាមញ្ញជាងនេះមួយកម្រិតទៀត៖


Ownership
Web3 ផ្តល់ឱ្យអ្នកនូវ digital asset ក្នុងទម្រង់ដែលមិនធ្លាប់មានពីមុនមក។ ឧទាហរណ៍: នៅក្នុង web2 ហ្គេម ប្រសិនបើអ្នកបានទិញ item នៅក្នុង account នោះនឹងភ្ជាប់នៅក្នុង account របស់អ្នក។ វានឹងអាចបាត់បង់ឬ ផ្លាស់ប្តូរពីសំណាក់អ្នកបង្កើតហ្គេម។ តែចំពោះ Web3 វិញ item ទាំងនោះ Web3 នឹងអនុញ្ញាតឱ្យអ្នកគ្រប់គ្រងដោយផ្ទាល់ តាមរយ:ជា owership ក្នុងទម្រង់​ non-fungible-token (NFT) និង គ្មានអ្នកណាអាចផ្លាស់ប្តូរ item របស់អ្នកបានឡើយ ហើយអ្នកអាចធ្វើការ លក់ ឬ​ trade នៅក្នុងទីផ្សារបានទៀតផង។



Censorship Resistance
នៅក្នុង Web3 ទិន្នន័យរបស់អ្នក រក្សាទុកនៅក្នុង​ Blockchain។​ មានន័យថា វាផ្តល់ទំនុកចិត្តទៅលើទិន្នន័យរបស់អ្នក ហើយទិន្នន័យនោះនឹងមិនអាចលុបបាន ឬលួចកែប្រែដោយអ្នកណាម្នាក់ឬកាន់កាប់ដោយ អ្នកបង្កើត service នោះទេ។



Decentralized Autonomous Organizations (DAOs)
មិនមែនតែអ្នកអាចគ្រប់គ្រងទិន្នន័យរបស់អ្នកបាននោះទេ នៅក្នុង Web3 អ្នកថែមទាំងអាចគ្រប់គ្រង platform ដែលនឹងចូលរួមជាក្រុមបានផងដែរ។ ដោយអ្នកទាំងនោះ ចំណាយទុនជា Tokens ធ្វើជា​ Shares នៅក្នុងក្រុមហ៊ុន។ DAOs អនុញ្ញាតឱ្យយើងសម្របសម្រួល decentralized ownership នៃ platform មួយ និង ធ្វើការសម្រេចចិត្តពីអនាគតទៅលើផ្នែកណាមួយ។ DAOs ត្រូវបានប្រកាសឡើង តាមរយ:ការយល់ព្រមព្រៀងនៅលើ Smart Contract ដែលធ្វើការសម្រេចចិត្តដោយស្វ័យប្រវត្តិ បែប decentralized ទៅលើ pool នៃ resources(tokens)។ អ្នកប្រើប្រាស់ ប្រើ token ដើម្បី vote ទៅលើរបៀបនៃការចំណាយទាំងឡាយរបស់ resource ហើយ code នឹងស្វ័យប្រវត្តីនៃការដំណើរការណាមួយនៃ voting outcome។



Identity
ទូទៅ នៅពេលដែលអ្នកប្រើប្រាស់ platform services ច្រើនដូចជា Google, Facebook, Youtube, Reddit អ្នកនឹងប្រាកដជាចុះឈ្មោះគ្រប់ platform នោះ។ ប្រសិនបើអ្នកចង់ផ្លាស់ប្តូរ profile picture អ្នកនឹងត្រូវផ្លាស់ប្តូរគ្រប់ platform ផងដែរ។ តែទោះជាយ៉ាងណា អ្នកក៏អាចធ្វើការ sign-in ដោយប្រើប្រាស់ social account បានប៉ុន្តែវាជាបញ្ហានៃ censorship (platform ខ្លះតម្រូវឱ្យអ្នកជឿទុកចិត្តទៅលើគេ ហើយផ្តល់ឱ្យនូវពត៌មានផ្ទាល់ខ្លួនមួយចំនួន)។
Web3 បានដោះស្រាយបញ្ហានេះផងដែរ ដោយអាចឱ្យយើងមាន digital identity ភ្ជាប់ជាមួយ Ethereum address និង ENS profile។​ ប្រើប្រាស់ Ethereum address ដើម្បីផ្តល់ជា single login ទៅលើ platforms ជាច្រើនដែល secure, censorship-resistant និង anonymous។



Native Payments
ដូចដែលបានបញ្ជាក់ខាងលើម្តងរួចហើយដែរនោះ Web2 payment ត្រូវការពឹងផ្អែកទៅលើភាគីកណ្តាលដូចជា ធនាគារ ឬ សេវាផ្សេងៗ។ Web3 ប្រើប្រាស់ token សម្រាប់ការចាយវាយដោយផ្ទាល់តាម browser បានដោយមិនត្រូវការភាគីទី៣ឡើយ។ ឧទាហរណ៍: ការចាយវាយដោយប្រើប្រាស់​ token (ETH)
